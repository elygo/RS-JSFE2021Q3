Dynos

The Heroku Platform uses the container model to run and scale all Heroku apps.

A dyno is a virtual linux container used at Heroku.

Dyno is the term Heroku uses for its web servers. If your app scales, scales the number of dynos as well. Consequently, the price for the service changes too.

Cons

  • Higher prices compared to other vendors
  • If app is inactive 30 mins, they put it to sleep
  • Architectural limitations for bigger applications
  • The needed add-ons might be expensive
You can change your subscription from free to hobby $7 per dyno per month
Up to 1000 dynos per month

Deployment

  1. Create an account at Heroku
  2. Install the Heroku CLI (from devcenter)
    $ npm install -g heroku
  3. Navigate to your folder
  4. Login to the Heroku CLI
    $ heroku login
  5. Create an app on Heroku
    $ heroku create
  6. Deploy the code to Heroku
    $ git push heroku main
Ok, deploy it!
